## FAQ

Tabnine supports many common languages such as Python, Java, JavaScript, C++, C#, Ruby, and others. The exact list may vary depending on version and plan.

**2. Does Tabnine work in all common IDEs?**
Yes, Tabnine is compatible with popular editors and IDEs such as Visual Studio Code, IntelliJ IDEA, Sublime Text, Atom, and others.

**3. Is Tabnine secure with regard to data protection?**
Tabnine offers options for local models that do not send code to the cloud. This means code can be processed securely and privately.

**4. How does the free version differ from the paid ones?**
The free version offers basic features, while paid plans include advanced AI models, team features, and better support.

**5. Do you need an internet connection for Tabnine?**
An internet connection is required for cloud-based models. Local models also work offline.

**6. Can Tabnine automatically correct code too?**
Tabnine mainly suggests code completions, but does not offer automatic correction. Error detection and suggestions are part of the functionality, however.

**7. How easy is installation and setup?**
Installation is done through plugins or extensions for the respective IDE and is usually quick and straightforward.

**8. Does Tabnine support teamwork?**
Yes, paid plans allow synchronization of settings and models within a team for consistent suggestions.

**9. How should Tabnine be tested?**
Best with a small, real scenario from your own day-to-day work. Check whether the tool helps write recurring code patterns faster and reduce context switching, and whether the results can be used without much rework.

**10. What is the most common stumbling block with Tabnine?**
The most common stumbling block is starting too broadly. Before rollout, it should be clear which repositories, licenses, and data protection rules apply in the IDE; otherwise, the benefit is hard to evaluate.
